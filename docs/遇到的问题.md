# 遇到的问题

## return 0的位置会影响运行库的链接

在无人机的项目上，我写了两个特别的动态库，一个是c库一个cpp库，其中cpp库会引用c库的一些函数来实现自己的一些函数。

在我的主程序配置文件里面，我同时依赖了这两个库，编译的时候非常顺利，但是运行的时候却发现找不到c库。

阴差阳错发现return 0的位置会影响这个编译结果，在某些情况下，能够正常使用，但是在某些情况下的时候不可以。

问一下大模型，即可只，写的程序有的部分会不可达，汇编的时候会扔掉。

因此，当我主程序只调用cpp库的时候，就会出现运行时链接失败。

但是当我调用cpp库以及c库的时候，则不会失败。

通过ldd命令查找对应的依赖链接，如`ldd ./bin/main`即可查看main程序的依赖动态库是什么，也可以查看动态库的动态库是什么。

```shell
# 这个是主程序没有明显调用c库函数的链接情况
anvidia@ubuntu:~/aaa_mzz/official-Payload-SDK-3.11.0$ ldd ./bin/main |grep samp
        libsampleCppPart.so => /home/anvidia/aaa_mzz/official-Payload-SDK-3.11.0/./bin/../build/linux/arm64/release/libsampleCppPart.so 
        libsampleCPart.so => not found
# 这个是主程序显式调用c库函数的链接情况
anvidia@ubuntu:~/aaa_mzz/official-Payload-SDK-3.11.0$ ldd ./bin/main |grep samp
        libsampleCppPart.so => /home/anvidia/aaa_mzz/official-Payload-SDK-3.11.0/./bin/../build/linux/arm64/release/libsampleCppPart.so 
        libsampleCPart.so => /home/anvidia/aaa_mzz/official-Payload-SDK-3.11.0/./bin/../build/linux/arm64/release/libsampleCPart.so 
```

结果发现main依赖c库以及cpp库，这是非常正常的，因为main依赖cpp库，但是cpp库又依赖c库，所以依赖关系传下来了。

但是问题是c库的指向路径是空的，再查看`ldd ./build/.../cpplib.so`发现其依赖于c库，但是指向路径也是空的。。。

```shell
# 问题在这，cpp库也找不到c库：
anvidia@ubuntu:~/aaa_mzz/official-Payload-SDK-3.11.0$ ldd ./build/linux/arm64/release/libsampleCppPart.so |grep samp
        libsampleCPart.so => not found
```

这是为什么，因为xmake默认设置比较保守，其认为随意把cpp库的依赖就是c库的路径暴露给主程序是比较危险的，所以传下来一个空的地址。

> 在 XMake 中，通常 `add_deps("sampleCPart")` 会自动处理链接顺序和传递性依赖。但是，**RPATH 不会自动默认设置为 `$ORIGIN`**（出于安全考虑，默认通常是空或系统路径）。因此，对于非系统安装的自定义动态库依赖，**必须手动添加 `add_rpathdirs("$ORIGIN")`**。

因此只需要在构建动态库目标的时候，加上一行指令即可。

如下：

```lua
-- 添加子xmake依赖
-- 添加三方库依赖
-- 一些路径环境变量
target("sampleCPart")
    add_defines("SYSTEM_ARCH_LINUX",{public=true})
    -- 基础属性设置
    add_rpathdirs("$ORIGIN") -- $ORIGIN 代表动态库文件自身所在的目录，传递给引用该库的库###################
    set_kind("shared")
    -- 添加源文件
    add_files("./src/**.c")
    -- 添加头文件
    add_includedirs("./src/",{public=true})

    -- 链接子xmake依赖
    -- 链接三方库依赖
    -- 添加运行库依赖
```

## 在无人机项目中，就算逻辑是对的，但还是调不出视频流

这个原因纯粹是cmake转xmake不太熟练，虽然我把所有的系统库、三方库、运行库都加上去了，但还是忽略了宏，cmake在查找的时候会定义一些宏变量，比如FFMPEG_INSTALLED，在代码中，有很多`#ifdef FFMPEG_INSTALLED` 的语句，所以当我没有定义这些宏变量的时候，就会导致这些语句没有运行，导致出错。
